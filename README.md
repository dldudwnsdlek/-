# 공급망 의사결정 모델 — 스코어링 프로토타입

환경기초시설(소각·하수처리·바이오가스 등) 관급자재 발주 시, 공급업체의 **낙찰 실적**과 **사회적 책임 지표**를 함께 고려해 발주 실무자의 의사결정을 지원하는 AI 기반 공급망 추천 모델의 프로토타입입니다.

| 항목 | 내용 |
|------|------|
| 대상 공모전 | 부울경환경본부 환경시설관리처 시설사업1부 — 지정분석 과제 |
| 문서 버전 | v0.1 (팀 회의용) |
| 작성일 | 2026-04-23 |
| 분석 대상 | 환경시설 경험 업체 4,003개 |

## 1. 프로젝트 개요

공모전 PDF의 입출력 사양(`입력 발주계획 → 출력 공급망 순위`)을 준수하면서, **AI 클러스터링**과 **해석 가능한 가중합 스코어링**을 결합한 공급망 추천 파이프라인을 제공합니다.

### 설계 원칙

- **AI는 클러스터링에서 책임지고, 점수는 해석 가능한 가중합으로 책임진다.**
  발주 실무자가 "왜 이 업체가 1위인가?"를 물었을 때, 점수 식이 명료해야 설명력이 확보됩니다.
- **각 Stage는 단일 책임 함수로 분리** — 가중치·k·임계값 변경 시 한 곳만 수정하면 됩니다.
- **적합성(Fitness)은 클러스터링에서 제외** — 발주 종속 변수이므로 추천 시점에 동적으로 계산해, 클러스터는 '업체의 본질'만 반영합니다.

## 3. 아키텍처 — 4단계 파이프라인

```
┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
│ Stage 1         │   │ Stage 2         │   │ Stage 3         │   │ Stage 4         │
│ 피처 엔지니어링 │ → │ 클러스터링      │ → │ 스코어링        │ → │ 추천 API        │
│ (10개 피처)     │   │ (KMeans k=5)    │   │ (다목적 가중합) │   │ (top-N 출력)    │
└─────────────────┘   └─────────────────┘   └─────────────────┘   └─────────────────┘
```

| Stage | 함수 | 역할 |
|-------|------|------|
| 1 | `build_features()` | 실적·사회적·안정성 10개 피처 생성 |
| 2 | `fit_clusters()` / `_assign_personas()` | KMeans 군집화 + 페르소나 5종 자동 라벨링 |
| 3 | `compute_scores()` | 카테고리별 점수 정규화 후 가중합 |
| 4 | `recommend_suppliers()` | 발주 키워드·예산 기반 top-N 추천 |


## 5. 피처 엔지니어링 (10개)

### A. 실적 (Performance) — 3개

| 피처 | 산식 | 의도 |
|------|------|------|
| `실적_건수_log` | `log1p(환경_낙찰건수)` | 극단값 완화 (단건 업체 60.3% vs 60건 초과 상위 업체) |
| `실적_금액_log` | `log1p(환경_낙찰금액합)` | 동일 |
| `실적_최신성` | `1 - (today - 환경_최근낙찰일).days / 1095` | 3년 내 활동 가중 (0~1 clip) |

### B. 사회적 책임 (Social) — 5개

| 피처 | 산식 |
|------|------|
| `자활용사촌_num`, `여성기업_num`, `장애인기업_num`, `사회적기업_num` | YN → 0/1 |
| `사회적책임_지표합` | 입력 CSV의 기존 컬럼 (0~4) |

### C. 안정성 (Stability) — 2개

| 피처 | 산식 | 의도 |
|------|------|------|
| `환경_집중도` | `환경_낙찰건수 / 전체_낙찰건수` | 환경시설 전문성 |
| `실적_지속성` | `(환경_낙찰건수 ≥ 5) → 1` | 1회성 vs 지속 공급 구분 |

### D. 적합성 (Fitness) — 추천 시점 동적 계산

| 피처 | 산식 |
|------|------|
| `품목_경험_norm` | `log1p(품목_낙찰건수) / log1p(max)` |
| `예산_적합도` | `1 - |log(평균낙찰금액) - log(추정금액)| / 5` |

## 6. 클러스터링 전략

- **알고리즘:** `KMeans(n_clusters=5, random_state=42, n_init=10)`
- **전처리:** `StandardScaler`로 10개 피처 표준화
- **품질 지표:** `silhouette_score` ≥ 0.20 목표 (현재 **0.441** 달성)

### 페르소나 자동 라벨링 규칙

군집 중심값의 **실적 점수 순위**와 **사회적 점수 순위** 조합으로 자동 분류합니다.

| 페르소나 | 실적 | 사회적 | 의사결정 활용 | 현재 분포 |
|---------|------|--------|---------------|----------|
| **우수공급군** | 상위 2 | 상위 2 | 최우선 추천 | 2개 |
| **대형실적군** | 상위 2 | 하위 | 균형 필요 — 사회적기업 보조 병행 | 493개 |
| **잠재공급군** | 하위 | 상위 2 | 사회적 가치 달성용 우선 | 124개 |
| **신규/소규모군** | 최하위 | - | 검증 후 시범 발주 | 750개 |
| **일반공급군** | 중위 | 중위 | 백업 풀 | 2,634개 |

## 7. 스코어링 공식

### 카테고리별 점수 (각 0~1 정규화)

```
점수_실적   = 0.5 · norm(실적_건수_log) + 0.3 · norm(실적_금액_log) + 0.2 · 실적_최신성
점수_사회적 = 사회적책임_지표합 / 4
점수_안정성 = 0.5 · 환경_집중도 + 0.5 · 실적_지속성
점수_적합성 = 0.5 · 품목_경험_norm + 0.5 · 예산_적합도    (추천 시점)
점수_리스크 = (부정당제재 시 1, 아니면 0)                  (현재 0 placeholder)
```

### 종합점수 (가중합)

```
종합점수 = 0.30 · 점수_실적
        + 0.30 · 점수_사회적
        + 0.15 · 점수_안정성
        + 0.20 · 점수_적합성
        - 0.05 · 점수_리스크
```

**가중치 결정 근거**
- 사회적 0.30 — 공모전 최우선 목표 (실적과 동일 비중)
- 실적 0.30 — 공급 능력 검증
- 적합성 0.20 — 발주 품목 매칭은 필수 조건
- 안정성 0.15 — 보조 지표
- 리스크 -0.05 — 부정당제재 확보 시 하드 필터로 전환 예정

> 가중치는 `WEIGHTS` dict로 분리 — 회의 후 한 줄만 수정하면 튜닝 가능.

```python
print(rec)
```

## 11. 한계 및 향후 과제

### 현재 가정·한계

1. **세부품명번호(10-digit)** 가 원본 CSV에 없어서 키워드 매칭으로 대체 → 나라장터 추가 API 호출 필요
2. **부정당제재 데이터** 미확보 → 리스크 점수가 현재 0 placeholder
3. **자활용사촌 매칭이 2건뿐** — 업체명 정규화 한계, 사업자번호 기반 조인 필요
4. **다중지표 보유 업체가 1.1%뿐**이라 우수공급군이 2개로 좁음

### 회의 논의 대상

1. 가중치 (실적 0.30 / 사회적 0.30 / 안정 0.15 / 적합 0.20 / 리스크 -0.05) 유지 여부
2. 페르소나 5개 라벨의 명칭과 분류 규칙
3. 추가 데이터 수집 우선순위 — 부정당제재 vs 사회적협동조합 vs 세부품명번호
4. 대시보드 형태 — Streamlit 인터랙티브 vs 정적 HTML 리포트

### 후속 과제

- 세부품명번호 컬럼 수집 (나라장터 추가 API)
- 부정당제재 이력 → 하드 필터화
- 사회적협동조합·장애인표준사업장·창업기업 데이터 보강
- 클러스터링 알고리즘 비교 (GMM·DBSCAN)
- 가중치 자동 튜닝 (그리드 서치 + 백테스트 hit rate 최대화)
- Streamlit/Plotly Dash 인터랙티브 대시보드
- 실시간 데이터 파이프라인 (스케줄러 + DB)
