## 1. 프로젝트 배경 및 목적

환경기초시설(소각·하수처리·바이오) 설치 사업의 관급자재 구매 시, 공급업체의 **사회적 책임 지표**와 **낙찰 실적**을 함께 고려하여 발주 실무자의 의사결정을 지원하는 AI 모델을 개발한다.

**프로토타입 범위:** 데이터 적재 → 피처 엔지니어링 → 클러스터링 → 스코어링 → 추천 API까지 end-to-end 동작 검증. 대시보드/실시간 파이프라인은 후속 단계.

## 3. 시스템 아키텍처

```
┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
│ Stage 1         │   │ Stage 2         │   │ Stage 3         │   │ Stage 4         │
│ 피처 엔지니어링 │ → │ 클러스터링      │ → │ 스코어링        │ → │ 추천 API        │
│ (10개 피처)     │   │ (KMeans k=5)    │   │ (다목적 가중합) │   │ (top-N 출력)    │
└─────────────────┘   └─────────────────┘   └─────────────────┘   └─────────────────┘
```

설계 원칙:
- 각 Stage는 단일 책임 함수로 분리 → 가중치/k 변경 시 한 곳만 수정
- 적합성(Fitness) 피처는 클러스터링에서 제외하고 추천 시점에 동적 계산 → 클러스터는 발주와 무관한 업체 본질을 반영

## 5. 피처 엔지니어링 (10개)

### A. 실적 (Performance) — 3개
| 피처 | 산식 | 의도 |
|------|------|------|
| `실적_건수_log` | `log1p(환경_낙찰건수)` | 극단값 완화 |
| `실적_금액_log` | `log1p(환경_낙찰금액합)` | 동일 |
| `실적_최신성` | `1 - (today - 환경_최근낙찰일).days / 1095` (clip 0~1) | 3년 내 활동 가중 |

### B. 사회적 책임 (Social) — 5개
| 피처 | 산식 |
|------|------|
| `자활용사촌_num`, `여성기업_num`, `장애인기업_num`, `사회적기업_num` | YN → 0/1 |
| `사회적책임_지표합` | 입력 CSV의 기존 컬럼 (0~4) |

### C. 안정성 (Stability) — 2개
| 피처 | 산식 | 의도 |
|------|------|------|
| `환경_집중도` | `환경_낙찰건수 / 전체_낙찰건수` (clip 0~1) | 환경시설 전문성 |
| `실적_지속성` | `(환경_낙찰건수 ≥ 5) → 1` | 1회성 vs 지속 공급 구분 |

### D. 적합성 (Fitness) — 추천 시점 동적 계산, 클러스터링 제외
| 피처 | 산식 |
|------|------|
| `품목_경험_norm` | `log1p(품목 낙찰건수) / log1p(max)` |
| `예산_적합도` | `1 - |log(평균낙찰금액) - log(추정금액)| / 5` (clip 0~1) |

## 6. 클러스터링 전략

- **알고리즘:** KMeans (k=5), `random_state=42`, `n_init=10`
- **전처리:** `StandardScaler`로 10개 피처 표준화
- **k 선택 근거:** 페르소나 5종 사전 정의 + silhouette score로 검증
- **품질 지표:** `silhouette_score` (0.2 이상 목표)

### 페르소나 자동 라벨링 규칙

군집 중심값(centroid)을 보고 **실적 점수(낙찰건수+금액 log)** 와 **사회적 점수(지표합)** 의 순위 조합으로 자동 분류:

| 페르소나 | 실적 순위 | 사회적 순위 | 의사결정 활용 |
|---------|----------|------------|--------------|
| **우수공급군** | 상위 2 | 상위 2 | 최우선 추천 |
| **대형실적군** | 상위 2 | 하위 | 균형 필요 — 사회적기업 보조 추천 병행 |
| **잠재공급군** | 하위 | 상위 2 | 사회적 가치 달성용 우선 |
| **신규/소규모군** | 최하위 | - | 검증 후 시범 발주 |
| **일반공급군** | 중위 | 중위 | 백업 풀 |

## 7. 스코어링 공식

### 카테고리별 점수 (각 0~1로 정규화)
```
점수_실적   = 0.5·norm(실적_건수_log) + 0.3·norm(실적_금액_log) + 0.2·실적_최신성
점수_사회적 = 사회적책임_지표합 / 4
점수_안정성 = 0.5·환경_집중도 + 0.5·실적_지속성
점수_적합성 = 0.5·품목_경험_norm + 0.5·예산_적합도   (추천 시점)
점수_리스크 = (부정당제재 시 1, 아니면 0)            (현재 0 placeholder)
```

### 종합점수 (가중합)
```
종합점수 = 0.30·점수_실적
        + 0.30·점수_사회적
        + 0.15·점수_안정성
        + 0.20·점수_적합성
        - 0.05·점수_리스크
```

**가중치 결정 근거:**
- 사회적 0.30 — 공모전 최우선 목표 (실적과 동일 비중)
- 적합성 0.20 — 발주 품목 매칭은 필수 조건
- 안정성 0.15 — 보조 지표
- 실적 0.30 — 공급 능력 검증

> 가중치는 `WEIGHTS` dict로 분리하여 회의 후 조정 가능.

## 8. 추천 API 사양

```python
recommend_suppliers(
    품목_키워드: str,
    추정금액: float,
    df_clustered: DataFrame,
    raw_lookup: DataFrame,
    top_n: int = 10,
    cluster_filter: list[str] | None = None,
    region: str | None = None,
    require_social: bool = False,
) -> DataFrame
```

### 처리 절차
1. 품목 키워드로 raw_lookup에서 후보 사업자번호 추출
2. 후보별 품목 낙찰건수·평균금액 집계 → 적합성 피처 계산
3. 종합점수 재계산 (적합성 동적 주입)
4. 필터 적용 (페르소나/지역/사회적책임 강제)
5. 종합점수 내림차순 → top-N 반환 (PDF 출력 사양)

## 9. 검증 계획

| # | 검증 항목 | 방법 | 합격 기준 |
|---|---------|------|----------|
| 1 | 클러스터 품질 | silhouette_score | ≥ 0.20 |
| 2 | 페르소나 해석 가능성 | centroid 수동 검토 | 의미 있는 라벨 5종 |
| 3 | 추천 적중률 (백테스트) | 2026 hold-out, top-10 hit rate | ≥ 30% |
| 4 | 가중치 민감도 | 가중치 ±0.1 변동, top-20 변동률 | < 30% |
| 5 | 사회적기업 노출도 | top-50 내 사회적기업 비율 | ≥ 25% |

## 10. 가정·한계·향후 과제

### 현재 가정
- 품목 식별을 키워드 매칭으로 대체 (세부품명번호 미확보)
- 부정당제재 데이터 미확보 → 리스크 점수 = 0
- 자활용사촌 매칭 2건만 확인됨 → 사업자번호 기반 보강 필요

### 향후 과제 (회의 논의 대상)
1. 세부품명번호 컬럼 수집 (나라장터 추가 API 호출)
2. 부정당제재 이력 수집 → 하드 필터화
3. 사회적협동조합·장애인표준사업장·창업기업 데이터 추가
4. 클러스터링 알고리즘 GMM/DBSCAN과 비교
5. Streamlit/Plotly Dash 인터랙티브 대시보드
6. 가중치 자동 튜닝 (그리드 서치 + 백테스트 hit rate 최대화)
